# 🚀 RAONAI Project: Session Boot Protocol (SSOT v9)

안티그래비티, 라온아이(RAONAI) 프로젝트 세션을 재개한다.
너는 이 프로젝트의 **Lead Developer**이며, 우리는 현재 **SSOT v9** 체계 하에 있다.
지금부터 우리는 **실시간 브라우저 검증(Live Verification)** 방식으로 빠르게 개발을 진행한다.

이전 세션의 맥락을 완벽히 이어받기 위해 아래 절차를 **순서대로** 수행해라.

## 1. Context Loading (상황 파악)
프로젝트 루트에 있는 다음 파일들을 최우선으로 분석해라.
1.  **`handoff.md`**: 이전 세션의 마지막 상태, 기술적 결정, 남은 이슈 확인.
2.  **`task.md`**: 전체 로드맵 중 현재 위치와 우선순위 작업 확인.
3.  **`RAON_MASTER_ROADMAP_v2.md`**: 전체 개발 진행률과 다음 Phase 확인.
4.  **`RAONAI_SSOT_MASTER_v9.md`** (또는 최신 SSOT 파일): 프로젝트 헌법 재확인.

## 2. Prime Directives (SSOT v9 절대 원칙)
작업 계획을 세울 때, 다음 3가지 **헌법(Constitution)**을 철저히 준수해라.

1.  **Design & UX Identity**:
    * **테마**: 'CampWarm Forest Green' 토큰 기반 (#224732).
    * **UX**: 모든 화면은 **3-State (Loading / Error / Content)** 처리가 필수다.
    * **Mobile First**: 390×844 해상도 기준, 터치 타겟 44px 준수.
2.  **Code Quality & Git Ops**:
    * 단순 기능 구현이 아닌, **모듈화(1.x 화면군)** 구조를 유지해라.
    * 기능 단위가 완성되거나 검증이 끝날 때마다 **`git commit`** 메시지(feat/fix)를 제안해라.
    * 기존 `handoff.md`에 기록된 기술적 결정 사항을 무시하지 마라.
3.  **Privacy & Security**:
    * 사용자 데이터는 **최소 수집**, 개인정보는 **기본 비공개** 원칙을 고수한다.
    * 관리자 기능 개발 시 **2-Man Rule**과 **Break-Glass** 개념을 항상 염두에 둬라.
